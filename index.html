Introductory Material

The main goal of PyLog is to formalise the entirety of Stephen G. Simpson's book Subsystems of Second-Order Arithmetic. PyLog combines minimalism with the philosophy of reverse mathematics. PyLog is based on natural deduction for classical first-order logic with an extension-forming operator.
The axioms of Z2 are interpreted as follows: 

0. Nat(x) v Set(x) 
1. Nat(x) -> ¬Set(x) 
2. Nat(0) & Nat(1) 
3. Nat(n) -> ¬((n + 1) = 0) 
4. (Nat(n) & Nat(m)) -> (((m + 1) = (n + 1)) -> (m = n)) 
5. Nat(m) -> ((m + 0) = m) 
6. (Nat(n) & Nat(m)) -> ((m + (n + 1)) = ((m + n) + 1)) 
7. Nat(m) -> ((m * 0) = 0) 
8. (Nat(n) & Nat(m)) -> ((m * (n + 1)) = ((m * n) + m)) 
9. Nat(m) -> ¬(m < 0) 
10. (Nat(n) & Nat(m)) -> ((m < (n + 1)) <-> ((m < n) v (m = n))) 
11. (x ε X) -> (Nat(x) & Set(X)) 
12. ((0 ε X) & ∀n.((n ε X) -> ((n + 1) ε X))) -> ∀n.(n ε X) 

n ε { x: form(x)}
-----------------
Nat(n) & form(n)


Nat(n) & form(n)
-----------------
n ε { x: form(x)}

The comprehension scheme is converted into the two rules above which are similar to ClassElim and ClassInt used to formalise Kelley-Morse set theory. Example of a simple proof:

1. Nat(0) & Nat(1)  AxInt 
2. 0 = 0  Identity 
3. Nat(0)  AndElimL 1
4. Nat(n) -> ¬((n + 1) = 0)  AxInt 
5. ∀n.(Nat(n) -> ¬((n + 1) = 0))  ForallInt 4
6. Nat(0) -> ¬((0 + 1) = 0)  ForallElim 5
7. ¬((0 + 1) = 0)  ImpElim 3 6
8. ¬((0 + 0) = 0)  EqualitySub 7 0
9. Nat(m) -> ((m + 0) = m)  AxInt 
10. ∀m.(Nat(m) -> ((m + 0) = m))  ForallInt 9
11. Nat(0) -> ((0 + 0) = 0)  ForallElim 10
12. (0 + 0) = 0  ImpElim 3 11
13. ¬(0 = 0)  EqualitySub 8 12
14. ┴  ImpElim 2 13
15. ¬(1 = 0)  ImpInt 14 Qed

Formal Set Theory

This project is to formalise and check the presentation of elementary axiomatic set theory found in the book General Topology by John Kelley (Van Nostrand, 1955). The section on set theory is found on pp.250-281. This axiomatic system is close to the von Neumann-Bernays-Gödel one and is sometimes called Kelley-Morse set theory.

PyHott

PyHott is a sister program of PyLog. It is based on homotopy type theory as presented in the apendix of the 'Hott Book'. In Univalent projects using Agda, type checking is automatic. PyHott is just a minimalist type inference assistant in the vein of PyLog. Thus given a typing of a term it will not tell you if it is correct. It will only tell you if a given type inference of such a typing is correct.
